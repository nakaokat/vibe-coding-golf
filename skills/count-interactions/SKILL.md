# Interaction Counter

タスクの開始から完了までの会話を分析し、人間の介入回数を集計・報告する。

## 目的

「1プロンプトでタスクを完了できるか」を測定する。人間の介入が少ないほど、初期プロンプトの品質とパーミッション設定が最適化されていることを示す。理想は **人間のメッセージ1回（初期指示のみ）+ ツール承認0回** である。

## 集計手順

会話履歴を先頭から順に読み、以下を分類・カウントする。

### Step 1: 人間のメッセージを分類する

会話中の人間のメッセージを以下のカテゴリに分類する:

| カテゴリ | 説明 | 例 |
|---|---|---|
| **初期指示** | タスクの最初の依頼メッセージ | 「デッドコードをチェックしてissueにして」 |
| **追加指示・判断** | Claudeの質問への回答や方針決定 | 「2つのissueを作成する」 |
| **確認・質問** | 進捗確認や成果物への質問 | 「全てのタスクは完了したか？」 |

### Step 2: ツール承認回数を推定する

Claude Code では、allowlist に含まれないツール呼び出しは人間の承認が必要になる。会話履歴から正確に自動承認/手動承認を区別することは難しいため、以下のルールで推定する:

- **Claudeの1つのレスポンス内で連続実行されたツール群** → 1回の承認イベント（またはまとめて自動承認）
- **Claudeのレスポンスが分断され、次のレスポンスで再度ツールを呼んでいる場合** → その間に承認が発生したと推定
- **Agent ツール（サブエージェント）のツール呼び出し** → 親の承認に含まれるためカウントしない

つまり「Claudeのターン（レスポンス）の数 - 1」が、ツール承認でClaudeの実行が中断された回数の概算になる。ただし、Claudeが自発的に複数ターンに分けた場合もあるため「推定」と明記する。

### Step 3: 報告する

以下のフォーマットで報告する:

```
## Interaction Report

| 指標 | 回数 |
|---|---|
| 人間のメッセージ数 | X 回 |
| - 初期指示 | X 回 |
| - 追加指示・判断 | X 回 |
| - 確認・質問 | X 回 |
| ツール承認（推定） | 約 X 回 |
| **合計インタラクション** | **X 回** |

### 詳細

1. [初期指示] 「（メッセージの要約）」
2. [ツール承認] （ツール名と概要）× N回
3. [追加指示] 「（メッセージの要約）」
...

### 改善提案

- ...
```

### Step 4: 改善を提案する

集計結果に基づき、次回同じタスクを1プロンプトで完了するための具体的な提案を行う:

- **追加指示・判断が発生した場合**: Claudeが質問した内容を特定し、初期プロンプトに含めるべき情報を提案する。改善版プロンプトの例を示す。
- **ツール承認が多い場合**: 頻出するコマンドパターンを特定し、settings.json の allowlist に追加すべきパーミッションを `/update-config` の形式で提案する。
- **確認・質問が発生した場合**: CLAUDE.md やプロンプトに事前に含められた判断基準を提案する。

## 注意事項

- このスキルは会話の最後（タスク完了後）に呼び出されることを想定している
- スキル自体の呼び出し（「やりとりを数えて」等）はカウント対象に含めない
- 推定値には必ず「推定」と明記し、正確性の限界を伝える
